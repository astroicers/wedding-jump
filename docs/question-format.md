# 題目格式說明

## CSV 檔案位置

`server/data/questions.csv`

## 格式類型

Wedding Jump 支援兩種題目格式：

### 1. O/X 二選一格式（原版）

```csv
題目,倒數時間,正確答案,分數
"新郎最喜歡吃牛排？",30,O,100
"新娘是獨生女？",25,X,150
"他們的第一次約會是在咖啡廳？",30,O,100
```

**欄位說明：**

| 欄位 | 類型 | 說明 |
|------|------|------|
| 題目 | 字串 | 問題文字，可用引號包裹 |
| 倒數時間 | 數字 | 回答時間（秒） |
| 正確答案 | O/X | O 表示正確，X 表示錯誤 |
| 分數 | 數字 | 答對獲得的分數 |

### 2. A/B/C/D 四選一格式（v2）

```csv
題目,選項A,選項B,選項C,選項D,倒數時間,正確答案,分數
"新郎最喜歡的食物是？","牛排","壽司","披薩","義大利麵",30,A,100
"他們在哪裡相識？","大學","工作","朋友介紹","交友軟體",25,C,150
"新娘的職業是？","工程師","設計師","老師","醫生",30,B,100
```

**欄位說明：**

| 欄位 | 類型 | 說明 |
|------|------|------|
| 題目 | 字串 | 問題文字 |
| 選項A | 字串 | 第一個選項 |
| 選項B | 字串 | 第二個選項 |
| 選項C | 字串 | 第三個選項 |
| 選項D | 字串 | 第四個選項 |
| 倒數時間 | 數字 | 回答時間（秒） |
| 正確答案 | A/B/C/D | 正確選項 |
| 分數 | 數字 | 答對獲得的分數 |

---

## 自動偵測格式

系統會根據 CSV 欄位自動偵測格式：

- 有 `選項A` 欄位 → 四選一模式
- 無 `選項A` 欄位 → 二選一模式

## 注意事項

1. **編碼**: CSV 檔案請使用 UTF-8 編碼
2. **引號**: 含逗號的文字需用引號包裹
3. **換行**: 每題一行，不支援多行題目
4. **驗證**: 系統會自動驗證格式，無效題目會被跳過
5. **快取**: 題目載入後會快取 60 秒

## 題目設計建議

### 難度分級

| 難度 | 倒數時間 | 分數 | 說明 |
|------|----------|------|------|
| 簡單 | 15-20 秒 | 50 分 | 大家都知道的事實 |
| 中等 | 25-30 秒 | 100 分 | 需要一點思考 |
| 困難 | 35-45 秒 | 150-200 分 | 只有親近的人知道 |

### 題目類型建議

- **關於新郎**: 喜好、習慣、職業、興趣
- **關於新娘**: 喜好、習慣、職業、興趣
- **關於兩人**: 相識經過、約會故事、共同興趣
- **婚禮相關**: 婚禮細節、蜜月計畫
- **趣味題**: 笑話、糗事、有趣回憶

### 建議題數

- 短遊戲: 10-15 題
- 標準遊戲: 15-25 題
- 長遊戲: 25-40 題
